# Task Management Guide

ragtag includes a built-in task management system powered by `@task` tags. Tasks are embedded directly in your plain text note files and managed through the `ragtag task` subcommand.

## How It Works

A task is simply a `@task(...)` tag in any plain text file. ragtag discovers and parses these tags just like any other tag, then provides specialized commands for creating, listing, and updating tasks.

Because tasks live inside your notes, you always have full context — the task sits right next to the relevant documentation, plan, or idea.

## Task Syntax

A task at minimum requires a `title` and `ttc_estimate`:

```
@task(title="Fix the bug", ttc_estimate=2, time_units="hours")
```

A fully-specified task with all attributes:

```
@task(
    id="a1b2c3d4e5f67890",
    pid="parent_task_id_hex",
    title="Implement feature X",
    description="Add support for feature X in the parser",
    owner="alice",
    status="active",
    priority=1,
    time_spent=3.5,
    ttc_estimate=8,
    ttc_actual=10,
    time_units="hours"
)
```

Trailing text after the closing `)` on the same line is captured as well.

## Task Attributes

| Attribute | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `id` | string | No* | (generated) | Unique 16-character hex identifier |
| `pid` | string | No | (none) | Parent task ID for hierarchical organization |
| `title` | string | **Yes** | — | Short task title |
| `description` | string | No | (none) | Longer description of the task |
| `owner` | string | No | `"me"` | Person responsible for the task |
| `status` | string | No | `"new"` | Current status (must be a valid keyword) |
| `priority` | integer | No | (none) | Priority level; `0` is highest/most urgent; must be a non-negative integer |
| `time_spent` | float | No | (none) | Time already spent on this task |
| `ttc_estimate` | float | **Yes** | — | Estimated time to complete |
| `ttc_actual` | float | No | (none) | Actual time to complete (filled in after) |
| `time_units` | string | No | `"hours"` | Unit for time values: `hours`, `days`, or `weeks` |

*\* `id` is auto-generated by the `create` command but required for `set-*` commands to find the task.*

## Status Categories

Task statuses are grouped into five categories, each with a set of recognized keywords and a terminal color:

| Category | Color | Default Keywords |
| --- | --- | --- |
| **Done** | Green | `done`, `finished`, `complete`, `completed` |
| **Active** | Yellow | `active`, `underway`, `working`, `wip` |
| **Blocked** | Red | `blocked` |
| **Abandoned** | Orange | `abandoned`, `deleted`, `removed`, `dead` |
| **Inactive** | Gray | `inactive`, `pending`, `new` |

Only recognized status keywords are accepted. Attempting to set an unrecognized status produces an error.

> **Note:** Colors are automatically disabled when output is piped to another command or redirected to a file. Use `--no-color` to explicitly disable colors, or set the `NO_COLOR` environment variable.

### Priority Color

* Priority `0` is displayed in **red** (highest urgency)
* All other priority values are displayed without color

### Customizing Status Keywords

Status keywords can be customized in your [configuration file](configuration.md):

```yaml
tasks:
  status_keywords:
    done: ["done", "finished", "shipped"]
    active: ["active", "wip", "in-progress"]
    blocked: ["blocked", "waiting"]
    abandoned: ["abandoned", "cancelled"]
    inactive: ["inactive", "pending", "new", "backlog"]
```

## Commands

All task commands are accessed via `ragtag task <subcommand>`.

### `tasks create`

Generates a new `@task(...)` string and prints it to stdout. The output is meant to be copied into a note file.

**Non-interactive mode:**

```bash
ragtag task create \
    --title "Write documentation" \
    --ttc-estimate 4 \
    --time-units hours \
    --owner alice \
    --status new \
    --priority 1 \
    --description "Write user-facing docs for ragtag"
```

**Interactive mode:**

```bash
ragtag task create -i
```

In interactive mode, you are prompted for each field:

```
Title (required): Write documentation
TTC Estimate (required, numeric): 4
Description (leave blank to skip): Write user-facing docs
Owner (leave blank for default): alice
Status (leave blank for default):
Priority (leave blank to skip): 1
Time Units (leave blank for default):
Parent ID (leave blank to skip):
```

Required fields (`title`, `ttc_estimate`) cannot be skipped. Optional fields can be left blank to use defaults or omit.

**Flags:**

| Flag | Description |
| --- | --- |
| `--title <STR>` | Task title (required unless `-i`) |
| `--ttc-estimate <NUM>` | Time-to-complete estimate (required unless `-i`) |
| `--time-units <STR>` | Time units: `hours`, `days`, or `weeks` |
| `--description <STR>` | Task description |
| `--owner <STR>` | Task owner |
| `--status <STR>` | Initial status |
| `--priority <NUM>` | Priority level (`0` = highest) |
| `--pid <STR>` | Parent task ID |
| `-i`, `--interactive` | Launch interactive prompt |

### `tasks list`

Lists all tasks found in scanned files.

```bash
ragtag task list
ragtag task list --path ./notes
ragtag task list --filter status=active
ragtag task list --filter status!=done --sort priority
ragtag task list --show-attributes id,title,status,priority
ragtag task list --sort title --reverse
```

**Default output** shows each task on one line with these attributes: `id`, `status`, `title`, `description`.

```
notes/project.md id="a1b2c3d4e5f67890" status="active" title="Write docs" description="User-facing documentation"
```

**Flags:**

| Flag | Description |
| --- | --- |
| `--path <PATH>` | Search path (file or directory, default: `.`) |
| `--filter <EXPR>` | Filter tasks (repeatable). Format: `field=value`, `field!=value`, `field>value`, `field<value`, `field>=value`, `field<=value` |
| `--sort <FIELD>` | Sort results by field name (e.g., `priority`, `status`, `title`) |
| `--reverse` | Reverse sort order |
| `--show-attributes <LIST>` | Comma-separated list of attributes to display |

**Filter examples:**

```bash
# Only active tasks
ragtag task list --filter status=active

# Tasks not done, sorted by priority
ragtag task list --filter status!=done --sort priority

# High-priority tasks owned by alice
ragtag task list --filter priority=0 --filter owner=alice

# Tasks with estimate > 4 hours
ragtag task list --filter ttc_estimate>4
```

### `tasks summary`

Displays a table-like summary of all tasks, grouped by a field and with aligned columns.

```bash
ragtag task summary
ragtag task summary --group owner
ragtag task summary --group priority --sort title
ragtag task summary --path ./notes
```

**Output** shows tasks in grouped tables with headers:

```
=== Status: active ===
ID                Title              Owner   Status   Priority  Time Spent  TTC Est.  TTC Act.  Time Units
--                -----              -----   ------   --------  ----------  --------  --------  ----------
a1b2c3d4e5f67890  Design API         alice   active   0         3.5         8         -         hours
f0e1d2c3b4a59687  Write parser       bob     active   1         -           4         -         hours

=== Status: done ===
ID                Title              Owner   Status   Priority  Time Spent  TTC Est.  TTC Act.  Time Units
--                -----              -----   ------   --------  ----------  --------  --------  ----------
1122334455667788  Setup project      alice   done     2         2           2         2         hours
```

Status values are color-coded (green for done, yellow for active, red for blocked, orange for abandoned, gray for inactive). Priority `0` is displayed in red.

**Flags:**

| Flag | Default | Description |
| --- | --- | --- |
| `--path <PATH>` | `.` | Search path (file or directory) |
| `--group <FIELD>` | `status` | Group tasks by field: `status`, `owner`, or `priority` |
| `--sort <FIELD>` | — | Sort tasks within each group by any task field name |

### `tasks set-status`

Updates a task's status in-place in its source file.

```bash
# Non-interactive: provide status on the command line
ragtag task set-status --id a1b2c3d4e5f67890 --status done

# Interactive: omit --status to be prompted
ragtag task set-status --id a1b2c3d4e5f67890
```

In interactive mode, the current task details are printed first, then you are prompted for the new status.

After updating, a multi-line detail listing of the task is printed. Optional fields that have no value are omitted from the output.

```
  title: Write documentation
  description: User-facing docs for ragtag
  id: a1b2c3d4e5f67890
  owner: alice
  status: done
  priority: 1
  ttc_estimate: 4
  time_units: hours
```

**Flags:**

| Flag | Description |
| --- | --- |
| `--id <ID>` | Task ID (required) |
| `--status <VALUE>` | New status value (interactive prompt if omitted) |
| `--path <PATH>` | Search path (default: `.`) |

### `tasks set-time`

Updates a task's `time_spent` value.

```bash
ragtag task set-time --id a1b2c3d4e5f67890 --time 6.5
```

**Flags:**

| Flag | Description |
| --- | --- |
| `--id <ID>` | Task ID (required) |
| `--time <VALUE>` | New `time_spent` value (interactive prompt if omitted) |
| `--path <PATH>` | Search path (default: `.`) |

### `tasks set-owner`

Updates a task's owner.

```bash
ragtag task set-owner --id a1b2c3d4e5f67890 --owner bob
```

**Flags:**

| Flag | Description |
| --- | --- |
| `--id <ID>` | Task ID (required) |
| `--owner <VALUE>` | New owner (interactive prompt if omitted) |
| `--path <PATH>` | Search path (default: `.`) |

### `tasks set-parent`

Updates a task's parent ID for hierarchical organization.

```bash
ragtag task set-parent --id a1b2c3d4e5f67890 --pid f0e1d2c3b4a59687
```

**Flags:**

| Flag | Description |
| --- | --- |
| `--id <ID>` | Task ID (required) |
| `--pid <VALUE>` | New parent task ID (interactive prompt if omitted) |
| `--path <PATH>` | Search path (default: `.`) |

## Workflow Example

1. **Create a task** and paste it into your notes:

    ```bash
    ragtag task create --title "Design API" --ttc-estimate 8 --priority 0
    ```

    Copy the printed `@task(...)` into your note file.

2. **Start working** — update the status:

    ```bash
    ragtag task set-status --id <task-id> --status active
    ```

3. **Track time** as you go:

    ```bash
    ragtag task set-time --id <task-id> --time 3.5
    ```

4. **Check progress** across all your notes:

    ```bash
    ragtag task list --filter status!=done --sort priority
    ```

5. **View a summary** of all tasks grouped by status:

    ```bash
    ragtag task summary
    ragtag task summary --group owner
    ```

6. **Mark complete** when finished:

    ```bash
    ragtag task set-status --id <task-id> --status done
    ```
